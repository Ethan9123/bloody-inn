# ML 优化电脑 AI — 自我对弈 + 进化策略 (ES)

这个目录用机器学习的方法优化游戏里电脑 AI 的决策权重。

## 为什么用 ES 而不是深度神经网络
游戏里 AI 每一步的决策本质是一组「加权计划评分」(刺杀 / 拉拢 / 扩张 / 客房服务 / 洗钱…)。
对这样一个轻量、纯静态网页的启发式 AI 来说，最有效又最实际的「机器学习」不是给浏览器塞一个深度网络
(需要大量训练语料、还要随页面下发模型、每步推理都跑 NN，得不偿失)，而是 **自我对弈 + 进化策略**：

- 用一个忠于游戏经济的**无头模拟器**，让「候选权重」与「基线权重」对弈成千上万局；
- 以**胜率**为适应度，用 `(μ+λ)-ES`（带高斯变异与 σ 退火）进化出更强的权重；
- 产物只是一组更好的**数字权重**，烘焙回游戏即可——保持游戏依旧零依赖、纯静态。

这是货真价实的机器学习（进化 / 元启发式优化），可离线复现、无外部服务。

## 用法
```bash
node ml/optimize_ai.js [seed] [generations] [gamesPerEval]
# 例：跑 4 个独立种子，各 45 代、每次评估 450 局
for s in 1 2 3 4; do node ml/optimize_ai.js $s 45 450; done
```
每次输出含 `trainWinrate`（训练胜率）与 `valWinrate`（用全新随机流复测 4000 局的独立验证胜率）。

## 诚实的结论（sim-to-real 差距）
- 方法有效：ES 能稳定进化出在模拟器里胜过基线的权重。
- **但**部分种子出现 ~98–100% 的离谱胜率——这是优化器在**钻模拟器的空子**（过拟合），而非学到真能迁移的策略。
- 因此我们**没有**直接采用这些极端权重，而是取**可信**的候选（约 68% 对基线胜率），把它做成游戏里一个**可选**难度
  「学习叔叔 (ML调优)」（不替换任何原有 AI，零回归风险）。
- 跨种子的一致信号（也确实符合真实牌理）：**更重拆台(deny)、更早存钱(launderAt↓)、早期猛扩张、后期狠抢**。

最优可信权重见 `best_weights.json`，已烘焙进 `script.js` 的 `AI_PROFILES.ml`。

> 注：模拟器只建模 AI 自己的抽象经济（抢劫/收租/客房服务/洗钱/拆台），不建模人类的手牌/别馆/埋尸全经济——
> 因为电脑 AI 本身就不用那套。要让结论更可信，下一步可提高模拟器保真度，或在真实对局中做大样本迁移验证
> （目前真实对局与 DOM 强耦合，难以大规模无头跑）。
